# Восстановление локальной версии до проблемного коммита

## Проблема

Файлы были удалены или изменены во время проблемного коммита/пуша. Нужно восстановить локальную версию до этого момента.

## Решения

### Вариант 1: Автоматический поиск и восстановление (рекомендуется)

Запустите:
```bash
scripts/bat/find_and_restore_commit.bat
```

Этот скрипт:
1. Найдёт все коммиты с нужными файлами
2. Покажет список последних коммитов
3. Позволит выбрать коммит для восстановления
4. Восстановит все файлы из выбранного коммита

### Вариант 2: Откат к предыдущему коммиту

Если нужно просто откатиться на один коммит назад:

```bash
scripts/bat/reset_to_before_push.bat
```

Этот скрипт предложит:
- **Soft reset** - откатит коммит, но сохранит изменения в staging area
- **Hard reset** - полностью откатит коммит и удалит все изменения

### Вариант 3: Вручную через cmd

1. Найдите нужный коммит:
```bash
git log --all --oneline -- src/api/wb_catalog_api.py
```

2. Восстановите файлы из конкретного коммита:
```bash
git checkout [HASH] -- src/api/wb_catalog_api.py
git checkout [HASH] -- parse_brands.py
git checkout [HASH] -- brands_config.json
# и т.д.
```

3. Или откатитесь к предыдущему коммиту:
```bash
git reset --soft HEAD~1   # Сохраняет изменения
# или
git reset --hard HEAD~1   # Удаляет изменения
```

## Важно

- **Soft reset** - безопаснее, сохраняет ваши изменения
- **Hard reset** - удаляет все изменения, используйте осторожно
- После восстановления проверьте статус: `git status`
