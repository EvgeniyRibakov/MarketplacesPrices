# Восстановление удалённых файлов

Файлы были удалены во время rebase операции. Есть два способа восстановления:

## Способ 1: Восстановление из git истории (быстро)

Запустите один из скриптов:

**Батч-файл:**
```bash
scripts/bat/restore_files.bat
```

**Python скрипт:**
```bash
python scripts/restore_files.py
```

Эти скрипты попытаются восстановить файлы из последнего коммита в git истории.

## Способ 2: Пересоздание файлов

Если восстановление из git не сработало, файлы нужно пересоздать.

## Список файлов для восстановления:

1. `src/api/wb_catalog_api.py` - модуль для работы с внутренним API
2. `src/parsers/wb_parser.py` - парсер с методом parse_brand_catalog()
3. `src/utils/logger.py` - утилита для логирования
4. `parse_brands.py` - скрипт для парсинга брендов
5. `run_parse_brands.bat` - батч-файл для запуска
6. `run_parse_brands.ps1` - PowerShell скрипт для запуска
7. `brands_config.json` - конфигурация брендов
8. `docs/README_BRANDS_PARSING.md` - документация
9. `docs/GIT_INSTRUCTIONS.md` - инструкции по git

## Что делать дальше:

1. Попробуйте запустить `scripts/bat/restore_files.bat`
2. Если не сработало - скажите мне, и я пересоздам все файлы
3. После восстановления запустите `scripts/bat/fix_rebase_and_push.bat` для правильного коммита
